# PAST GUARDIANS - Quick Start Instructions
## Step-by-Step Setup for Unity + Claude MCP

---

## STEP 1: CREATE UNITY PROJECT

Open Unity Hub and create new project:
- Name: PastGuardians
- Template: 3D Mobile
- Version: Unity 2022.3 LTS or Unity 6

---

## STEP 2: INSTALL PACKAGES

Open Window > Package Manager and install:

1. AR Foundation (5.x)
2. ARCore XR Plugin
3. ARKit XR Plugin  
4. Netcode for GameObjects
5. TextMeshPro (usually included)

## STEP 6: START CODING WITH CLAUDE

Open Claude (via MCP, Claude.ai, or Claude Code).

Copy this initial prompt:

```
I'm building "Past Guardians" - a mobile AR game where players tap the sky to send time-displaced intruders back to their home planet.

I have Unity 2022.3 LTS with AR Foundation installed.

Please create IntruderData.cs - a ScriptableObject that defines intruder types with:
- intruderId (string)
- displayName (string)  
- era (enum: Prehistoric, Mythological, HistoricalMachines, LostCivilizations, TimeAnomalies)
- sizeClass (enum: Tiny, Small, Medium, Large, Boss)
- baseTapsRequired (int, range 30-500)
- stabilityRegenRate (float)
- baseAltitude (float, 50-1000 meters)
- tapWindowSeconds (float)
- movementPattern (enum: Hover, Drift, Circle, Escape, Approach)
- portalColor (Color)
- codexDescription (string)

Include [CreateAssetMenu] attribute and all enums.
```

## STEP 8: TESTING SCENE SETUP

Create main scene with:

1. AR Session Origin
   - Add AR Camera (tag: MainCamera)
   - Add AR Session component

2. Game Manager (Empty GameObject)
   - Attach: ARSetupManager
   - Attach: LocationManager
   - Attach: PlayerDataManager
   - Attach: IntruderSpawner

## STEP 10: BUILD AND TEST

1. File > Build Settings
2. Add current scene
3. Switch Platform to Android or iOS
4. Build and Run on device

Test checklist:
- [ ] AR camera activates
- [ ] GPS location detected
- [ ] Looking up triggers sky mode
- [ ] Test intruder appears
- [ ] Taps register
- [ ] Laser renders
- [ ] Return animation plays

## TROUBLESHOOTING

**AR not working on device:**
- Check XR Plug-in Management settings
- Verify device supports ARCore/ARKit
- Check camera permissions in Player Settings

**Location not updating:**
- Add location permission to Android manifest
- iOS: Add location usage description in Player Settings

**Touch not detecting:**
- Use Input.touchCount for mobile
- Add Input.GetMouseButton(0) for editor testing

**Performance issues:**
- Reduce max visible lasers
- Use object pooling for effects
- Check LOD settings on intruder models
